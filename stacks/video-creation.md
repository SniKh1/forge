# Video Creation Stack Pack

## Scope
用于视频脚本、镜头拆解、素材组织、剪辑流程、发布准备和视频内容生产协同。

## Defaults
- 先明确视频目标、受众和发布场景，再进入素材和剪辑细节。
- 优先把脚本、镜头、素材和成片结构理顺，再做风格化加工。
- 所有视频任务都默认要考虑可复用素材和版本管理。
- 视频类任务不只看“能产出”，还要看后续迭代和复用成本。

## Typical Inputs
- 视频选题和内容 brief
- 脚本或镜头需求
- 素材整理、字幕、封面、发布说明
- 多平台视频内容改写与复用

## Preferred Skills
- `brainstorming`
- `doc-coauthoring`
- `media-processing`
- `ai-multimodal`
- `internal-comms`
- `code-review`

## Recommended MCP / Tools
优先使用内容处理和协同能力：
- `browser-use`
- `fetch`
- `memory`
- `mcp-management`

v1 先不绑定具体第三方视频平台生态，后续按实际使用频率补充。

## Output Shape
- 视频 brief
- 脚本/镜头结构
- 素材整理规则
- 发布准备说明
- 复用与版本迭代说明

## Validation Checklist
- 目标受众和平台是否明确？
- 脚本、镜头、素材之间是否对应清楚？
- 是否说明了可复用部分与需重做部分？
- 是否考虑字幕、封面、发布时间、版本管理？

## Collaboration Contract
- 与 `product-manager + product` 配合时，先定义内容目标和发布目标。
- 与 `ui-designer + design` 配合时，重点对齐封面、标题、视觉节奏。
- 与 `developer` 配合时，只在确实需要自动化或系统支持时进入实现层。
