# Image Generation Stack Pack

## Scope
用于图像生成、风格迭代、视觉资产制作、提示词管理和图像资产工作流。

## Defaults
- 先明确用途、风格边界和使用场景，再进入 prompt 和生成参数。
- 优先沉淀可复用的 visual direction，而不是每次从零试错。
- 图像生成任务默认需要考虑尺寸、裁切、输出格式和后续复用。
- 产物不只要“好看”，还要适配真实投放与协作流程。

## Typical Inputs
- 海报、封面、活动图、社媒图
- 视觉参考、风格要求、图像 prompt
- 图像素材整理、批量生成、变体对比
- 图像生成工作流或资产管理需求

## Preferred Skills
- `aesthetic`
- `canvas-design`
- `ai-multimodal`
- `media-processing`
- `brainstorming`
- `doc-coauthoring`

## Recommended MCP / Tools
优先使用本地和通用能力：
- `browser-use`
- `fetch`
- `memory`
- `mcp-management`

如果后续接入特定图像平台或模型生态，再以独立补充文档接入。

## Output Shape
- visual direction note
- prompt / variation record
- 产物尺寸与用途说明
- 复用与命名规则
- 输出与审校说明

## Validation Checklist
- 图像用途是否明确？
- 风格边界是否清楚，而不是只有模糊形容词？
- 输出尺寸、格式、裁切是否适配目标场景？
- 是否记录了可复用 prompt 和失败方向？

## Collaboration Contract
- 与 `ui-designer + design` 配合时，优先输出可稳定复用的视觉方向。
- 与 `product-manager + product` 配合时，先对齐业务用途和投放场景。
- 与 `developer` 配合时，只在资产流程、批处理或工具化需求明确时进入实现层。
