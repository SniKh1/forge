# Obsidian Markdown

Use this when the user is working in an Obsidian vault and the markdown needs to follow Obsidian conventions rather than generic Markdown only.

## Focus Areas

- Wikilinks like `[[Note]]`
- Embeds like `![[Note]]`
- Callouts like `> [!note]`
- Frontmatter properties and tags
- Obsidian note structure and vault-friendly linking

## Guidance

- Use wikilinks for internal vault references
- Use standard Markdown links for external URLs
- Prefer frontmatter for structured metadata
- Keep Obsidian-specific syntax valid and readable
